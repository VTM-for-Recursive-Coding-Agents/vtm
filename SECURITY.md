# Security Policy

## Supported versions

Security fixes are applied on the active development branch.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| released tags older than `main` | No commitment |

## Reporting a vulnerability

Do not open a public issue for a suspected security problem.

Report vulnerabilities privately through GitHub security advisories for this repository. If advisories are not enabled yet, use the repository maintainers' shared security contact channel instead of contacting an individual maintainer directly.

When reporting, include:

- the affected component or file path
- reproduction steps or a proof of concept
- impact assessment
- any suggested mitigation if you already have one

You should expect an acknowledgment within a few business days. This repository is an active research codebase, so some hardening gaps may already be documented but still worth reporting.
