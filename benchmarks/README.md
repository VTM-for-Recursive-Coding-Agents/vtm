# benchmarks

Purpose: checked-in benchmark inputs for the maintained retrieval, drift, and targeted coding study. This directory is for durable inputs, not run outputs.

Contents
- `manifests/`: versioned benchmark manifests consumed by `vtm-bench`.

Run outputs, task packs, workspaces, executor artifacts, `predictions.jsonl`, and normalized SWE-bench harness artifacts belong under the chosen runtime output directory, not back in this checked-in asset tree.

Maintained benchmark recipes live in [`../docs/benchmark-recipes.md`](../docs/benchmark-recipes.md).
